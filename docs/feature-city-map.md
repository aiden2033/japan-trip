# Фича: карта города для каждого города (City Map)

> Спека для отдельного агента-исполнителя. Цель — на странице каждого города
> показать карту со штырьками всех мест города, чтобы было видно географию:
> что рядом, а что через весь город. Сейчас есть только текстовый список.

## Контекст проекта (важные ограничения)

- Стек: **Vite + React 18 + TypeScript + Tailwind v3 + react-router (`HashRouter`)**.
- Деплой на **GitHub Pages**, `vite.config.ts` → `base: '/japan-trip/'`.
- **Нет бэкенда, нет рантайм-API, гид позиционируется как офлайн-friendly.**
  Любое решение по карте должно деградировать корректно без сети.
- Данные мест: `src/data/places.ts` (массив `Place[]`), тип в `src/data/types.ts`.
- Сейчас у `Place` для геолокации есть **только** `mapsQuery: string` (текст для
  ссылки на Google Maps). **Координат (lat/lng) нет** — их нужно добавить.
- Страница города: `src/pages/CityPage.tsx`. Места уже делятся на:
  `mainPlaces` (обычные), `cafePlaces` (`foodSpot === true`), `dayTripPlaces`
  (`isDayTrip === true`, сгруппированы по `dayTripGroup`). Есть фильтр по тегам
  (`activeTags`) — карта должна уважать активный фильтр.
- Цвета городов: `CITY_ACCENT` в `src/lib/tags.ts` (osaka=розовый, kyoto=фиолетовый,
  tokyo=синий) — использовать для штырьков/акцентов.

## Объём работ

### 1. Модель данных: добавить координаты

В `src/data/types.ts` расширить `Place`:

```ts
coords?: { lat: number; lng: number }; // опционально, для карты
```

Сделать **опциональным**, чтобы:
- сборка не падала, пока координаты не у всех мест;
- места без координат просто не показывались на карте (или показывались списком
  «без точки на карте»).

Заполнить `coords` для мест в `src/data/places.ts`. Координаты известных мест
можно проставить вручную (Дотонбори, Осакский замок, Фусими Инари, Сибуя и т.д.).
**Никакого рантайм-геокодинга** — координаты должны быть статичными (билд-тайм).
Дейтрипы (Нара, Химэдзи, Хаконэ, Камакура, Кавагутико) тоже имеют координаты, но
они далеко за городом — учесть при расчёте границ карты (см. ниже).

### 2. Библиотека карты

Рекомендация: **Leaflet** (`leaflet` + `react-leaflet`), тайлы OpenStreetMap.
Причины: бесплатно, без API-ключа, лёгкая, работает на статике.

Альтернатива без зависимостей: статичная SVG-схема с относительным
позиционированием точек — полностью офлайн, но без реальной картографии. Выбрать
Leaflet, если приемлема загрузка тайлов из сети; SVG — если офлайн критичен.

**Офлайн-нюанс:** тайлы OSM грузятся из сети. В офлайне карта будет серой, но
штырьки и подписи останутся (можно держать список мест рядом как фоллбэк).
Когда появится PWA (см. roadmap) — добавить runtime-cache для тайлов уже
посещённых областей. Сейчас просто показать аккуратный плейсхолдер при ошибке
загрузки тайлов.

### 3. UI на странице города

- Переключатель вида в шапке города: **«Список / Карта»** (две кнопки-таба).
  По умолчанию — список (как сейчас). Состояние можно держать в `useState` или в
  query-параметре hash (`#/kyoto?view=map`) для шеринга.
- В режиме карты:
  - штырьки всех мест города, цвет = `CITY_ACCENT[city]`;
  - дейтрипы — отдельным стилем штырька (амбер, как амбер-рамка дейтрипа сейчас);
  - тап по штырьку → поповер с мини-карточкой (фото-превью `PlaceImage`, `nameRu`,
    теги, тайминг, кнопка «Подробнее» → `/{city}/{slug}`);
  - **карта уважает активный фильтр по тегам** (`activeTags` из `CityPage`):
    штырьки отфильтрованных мест скрываются/гаснут;
  - авто-фит границ (`fitBounds`) по видимым точкам города; дейтрипы за городом
    либо включать в границы по кнопке «показать дейтрипы», либо выносить отдельно,
    чтобы карта не отъезжала на пол-Японии.
- Доступность: штырьки фокусируемы с клавиатуры, у поповера есть закрытие по Esc,
  у кнопок `min-h-[44px]` как в остальном UI, focus-visible кольца в стиле проекта.

### 4. Возможные синергии (не блокирующие)

- Когда появится **планировщик «Мой день»** — подсветить на карте точки текущего
  плана и нарисовать примерный маршрут между ними.
- Кнопка «🗺 маршрут по всем точкам города» → directions-ссылка Google Maps.

## Критерии готовности

- На странице каждого города работает переключатель «Список / Карта».
- Штырьки кликабельны, ведут на детальную страницу места.
- Карта реагирует на фильтр по тегам.
- Места без `coords` не ломают карту.
- Корректная деградация без сети (плейсхолдер вместо тайлов, штырьки/список живы).
- `npm run build` проходит без ошибок типов (tsc -b + vite build).
- Стиль (Tailwind, скругления, тени, акцентные цвета городов) консистентен с проектом.

## Шаги самопроверки

1. `npm run build` — типы и прод-сборка зелёные.
2. `npm run dev` — вручную проверить все три города, фильтр по тегам, тап по штырьку,
   переход на детальную страницу, поведение при оффлайне (DevTools → Offline).
